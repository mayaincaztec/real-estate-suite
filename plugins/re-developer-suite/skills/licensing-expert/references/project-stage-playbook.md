# Project Stage Playbook

Dùng reference này khi `licensing-expert` cần vào đúng mode theo **giai đoạn dự án**, thay vì chỉ theo loại issue rời rạc.

## 1. Pre-acquisition / screening
### Tập trung vào
- legal status cấp cao;
- title / land-chain red flags;
- quy hoạch và approval blockers rõ nhất;
- issue nào cần hỏi thêm trước khi đi sâu.

### Deliverable phù hợp
- legal status note
- recommendation memo
- legal question list

## 2. Active DD / confirmatory review
### Tập trung vào
- missing docs ảnh hưởng DD conclusion;
- permit / title / approval gaps;
- issue nào là deal breaker vs conditional item;
- blocker cần gửi `RE-HQ` để nhúng vào key findings.

### Deliverable phù hợp
- hybrid legal package
- permit gap list
- blocker memo

## 3. Pre-transfer / pre-commercialization
### Tập trung vào
- điều kiện chuyển nhượng dự án / tài sản;
- điều kiện mở bán / huy động vốn;
- approvals / consents còn thiếu;
- remedial steps để đạt transferability hoặc commercial readiness.

### Deliverable phù hợp
- blocker memo
- recommendation memo
- approval matrix

## 4. Execution / remedial phase
### Tập trung vào
- thủ tục nào cần chạy tiếp;
- owner nào ngoài legal cần làm gì;
- tài liệu hoặc approval nào là critical path;
- legal chỉ nên nói phần compliance path, không ôm program management.

### Deliverable phù hợp
- approval matrix
- permit gap list
- legal question list

## 5. Post-issue / dispute-sensitive situation
### Tập trung vào
- hồ sơ / hành vi nào có thể dẫn đến risk hậu kiểm, tranh chấp hoặc xử phạt;
- điểm nào cần phối hợp `legal-counsel` nếu dispute or contractual exposure đã xuất hiện.

### Deliverable phù hợp
- recommendation memo
- blocker memo
- issue note phối hợp với `legal-counsel`
